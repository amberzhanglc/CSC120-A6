What are your initial impressions of how `Unit Testing` affects your programming?

At first, unit testing made me think more carefully about how each method should behave before running the program. Writing tests helped me find bugs more quickly and understand problems in the code.

What worked, what didn't, what advice would you give someone taking this course in the future?

In this assignment, I was able to detect several problems in the provided code, such as incorrect constructor behavior and issues in methods. One challenge was figuring out how to design tests that would expose hidden bugs. Sometimes the code looked correct at first, but writing a test revealed that a method was ignoring a parameter or behaving incorrectly. My advice would be try to test both normal behavior and situations where the program should throw an exception.
